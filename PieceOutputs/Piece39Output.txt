Day 7 — Set operations from a spec
====================================

## Setup

Tags table created and seeded (Einstein + Churchill quotes tagged; Mandela + Seed untagged).
Collections 'classic' (Churchill) and 'modern' (Einstein + Churchill overlap) created.

## Query 1 — Authors with quotes but no tags (EXCEPT)

SELECT DISTINCT Author FROM Quotes WHERE IsDeleted = 0
EXCEPT
SELECT DISTINCT q.Author FROM Quotes q JOIN Tags t ON t.QuoteId = q.Id;

Result:
╭────────╮
│ Author │
╞════════╡
│ Seed   │
╰────────╯

Operator used: EXCEPT
Why: EXCEPT removes from the left set any author that appears in the right set.
Left set = all authors with active quotes. Right set = authors who have at least
one tagged quote. What remains are authors with quotes but zero tags.

## Query 2 — Authors in both classic and modern collections (INTERSECT)

SELECT DISTINCT q.Author FROM Quotes q
JOIN CollectionItem ci ON ci.QuoteId = q.Id
JOIN Collections c ON c.Id = ci.CollectionId
WHERE c.Name = 'classic'

INTERSECT

SELECT DISTINCT q.Author FROM Quotes q
JOIN CollectionItem ci ON ci.QuoteId = q.Id
JOIN Collections c ON c.Id = ci.CollectionId
WHERE c.Name = 'modern';

Result:
╭───────────╮
│  Author   │
╞═══════════╡
│ Churchill │
╰───────────╯

Operator used: INTERSECT
Why: INTERSECT keeps only rows that appear in both result sets.
Churchill quotes were added to both collections, so Churchill is the only
author present in both. Einstein appears only in modern; Mandela and Seed
appear in neither.

## Query 3 — Combined distinct tag list across classic and modern (UNION)

SELECT DISTINCT t.Tag FROM Tags t
JOIN CollectionItem ci ON ci.QuoteId = t.QuoteId
JOIN Collections c ON c.Id = ci.CollectionId
WHERE c.Name = 'classic'

UNION

SELECT DISTINCT t.Tag FROM Tags t
JOIN CollectionItem ci ON ci.QuoteId = t.QuoteId
JOIN Collections c ON c.Id = ci.CollectionId
WHERE c.Name = 'modern';

Result:
╭────────────╮
│    Tag     │
╞════════════╡
│ leadership │
│ resilience │
│ wisdom     │
╰────────────╯

Screenshot Link:
https://github.com/thinkbridge-thinkschool/Taneem_thinkschoolDay1/blob/day7/piece39-set-operations/PieceOutputs/Piece39OutputSS.png

Operator used: UNION
Why: UNION merges both result sets and removes duplicates automatically.
'leadership' and 'resilience' come from classic (Churchill); 'wisdom' comes
from modern (Einstein). UNION ALL would list duplicates if a tag appeared
in both; UNION deduplicates so each tag appears once.
