Day 5 — Azure Container Apps fundamentals
==========================================

## az commands

# Create resource group
az group create -n thinkschool-rg-taneem -l eastus

# Create Container Apps environment
az containerapp env create -n thinkschool-env -g thinkschool-rg-taneem -l eastus

# Show environment JSON
az containerapp env show -n managedEnvironment-thinkschoolrgta-8c22 -g thinkschool-rg-taneem

## What we learned

- A Container Apps Environment is a shared boundary — all apps inside
  share the same virtual network, logging, and scaling infrastructure
- Consumption workload profile = serverless, pay per use, scales to zero
- provisioningState: Succeeded confirms the environment is ready
- staticIp is the shared outbound IP for all apps in the environment
- Microsoft.App provider must be registered in the subscription before
  any Container Apps commands work
- Region matters — student subscriptions restrict which regions are allowed.
  Central India was blocked; East US worked.
