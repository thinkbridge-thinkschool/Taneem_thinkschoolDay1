Day 7 — Joins and CTEs at depth
================================

## SQL Query

WITH AuthorStats AS (
    SELECT
        Author,
        COUNT(*)  AS QuoteCount,
        MAX(Id)   AS LatestQuoteId
    FROM Quotes
    WHERE IsDeleted = 0
    GROUP BY Author
)
SELECT
    s.Author,
    s.QuoteCount,
    q.Text AS MostRecentQuote
FROM AuthorStats s
INNER JOIN Quotes q ON q.Id = s.LatestQuoteId
ORDER BY s.QuoteCount DESC
LIMIT 10;

## Why CTE over a correlated subquery?
A CTE is more efficient because it calculates the aggregation results only once and then reuses them for all authors.
A correlated subquery, on the other hand, runs again for every single row.
With a small dataset like 4 authors, the performance difference is hardly noticeable, but with 10,000 authors it becomes expensive because the database keeps repeating the same work thousands of times.
The CTE version is also easier to read and maintain.
The aggregation logic is written separately with a clear name, so the final query becomes cleaner and easier to understand.
