Day 10 — EF Core change tracker + AsNoTracking
===============================================

## Query variants

### Tracked (default)
var quotes = await db.Quotes.ToListAsync();

EF loads each row, creates a Quote object, takes a snapshot of its property values,
and registers it in the change tracker. On SaveChanges(), EF diffs current values
against the snapshot to generate UPDATE SQL.

### AsNoTracking
var quotes = await db.Quotes.AsNoTracking().ToListAsync();

EF loads each row and creates a Quote object but skips the snapshot and change
tracker registration entirely.

---

## Timing and allocation results (10,000 rows, 3 runs each)

Tracked reads (default):
  Run 1: 257ms | 9,373 KB allocated
  Run 2: 120ms | 9,285 KB allocated
  Run 3: 105ms | 9,262 KB allocated

AsNoTracking reads:
  Run 1:  66ms | 3,705 KB allocated
  Run 2:  29ms | 3,523 KB allocated
  Run 3:  70ms | 3,507 KB allocated

## Delta

Metric       | Tracked   | AsNoTracking | Saving
-------------|-----------|--------------|-------
Time (avg)   | ~161ms    | ~55ms        | ~66% faster
Allocations  | ~9,307 KB | ~3,578 KB    | ~62% less memory

---

## When NOT to use AsNoTracking

Do not use AsNoTracking when you intend to modify the entity — EF has no snapshot
to diff against and SaveChanges() will not generate an UPDATE statement.
